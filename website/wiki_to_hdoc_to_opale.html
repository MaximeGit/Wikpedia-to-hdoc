What is Hdoc?

Hdoc is a generic XML schema based on XHTML5 for documentary purpose.

If you have any questions regarding Hdoc format, please take a look at http://ics.utc.fr/hdoc/www/co/hdocConverter.html

What is Wikipedia to Hdoc?

This project aims at providing documentary community with a converter that goes from Wikipedia to the new Hdoc format. Once one obtained an Hdoc zip, it is possible to go from Hdoc to any other format the community will propose a converter for.

I am interested in an Opale scar file, is Wikipedia to Opale released?

The community has been working on a Hdoc to Opale converter. We were able to test it with several Hdoc zips obtained from Wikipedia to Hdoc converter and we think the result is quite good, though it still has to be improved. So yes, Wikipedia to Opale is possible (go to download and documentation tab for this).

How to use it?

First download the converter link above. Then follow the next steps.

Getting started

After downloading the archive containing the converter, unzip it wherever you want. Using a terminal go to the root of the downloaded folder (wikipediaToHdoc).

Then go to the ant directory.

Generating .hdoc, .scar and .html of a Wikipedia article

In this example, let's say we are going to generate hdoc, scar and html outputs of the english bread article from Wikipedia.

  1. Copy the URL of the Wikipedia file you want to convert. In our example, it is http://en.wikipedia.org/wiki/Bread .

  2. Choose a name for the outputs. For the english bread example, let's say we want our exports to be named bread_en.hdoc/.scar/.html, so we will choose "bread_en" as a name.

  3. You will have to run a command like the following:

    ant -buildfile wiki_to_opale_fetcher.ant -DwikipediaUrl "yourWikipediaUrl" -Dfilename yourFilename

    Red words are parameters you will have to change everytime you want to obtain an export:

    • yourWikipediaUrl is the Wikipedia URL you copied earlier.
    • yourFilename is the name of the output files.

    In the english bread article example, here is the command we would execute:

    ant -buildfile wiki_to_opale_fetcher.ant -DwikipediaUrl "http://en.wikipedia.org/wiki/Bread" -Dfilename bread_en

    This will generate for you: bread_en.hdoc, bread_en.scar, bread_en.html (and also bread_en.xml which is Wikipedia xhtml if you want to use it).

Obtaining your new exports

Exports are generated in a folder called result. result will be generated at the root of the wikipedia_to_hdoc folder.

One folder by export will be created.

In the bread example, exports will be generated in wikipedia_to_hdoc/result/bread_en.

Images and table are not copied

Some elements are not processed. Images and tables are currently not copied. This is why some problems while trying to export might occur depending on which page you want to export...

Random errors might occur

Wikipedia is a great tool: everyone can participe. However, it does not provide contributors with best practices that everyone follows. The result is a lot of different ways to write articles. This is why this converter might not handle some situations, and it might not be able to output some Wikipedia articles at its current state.
